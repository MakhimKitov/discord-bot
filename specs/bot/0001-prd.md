## Product decision (owner, 2026-06-24)

- A **simple utility Discord bot**, built from a **minimal baseline**.
- Keep the baseline deliberately small: the point of the experiment is the **maintenance
  loop** driven by the autonomous developer, so most of the work should happen *after*
  v1 lands, not in it.

## Tech stack (owner, 2026-07-02)

**Python 3.12 + `discord.py`**, `uv` for env/deps.

- The owner reads Python (Go was the alternative).
- Planned maintenance tasks include **audio features** (see backlog): `discord.py` has
  first-class voice support and the `yt-dlp` + FFmpeg path is the best-documented route
  to them. In Go, voice means hand-rolling Opus frame handling — the maintenance tasks
  would measure stack friction instead of the developer.
- Slash commands only (`app_commands`); no message-content intent — the bot never reads
  free-form user messages, which also narrows the untrusted-input surface (rule 3).

## Requirements (v1 baseline)

Functional — three utility slash commands, enough to establish the command pattern:

- **`/ping`** — round-trip latency check; replies with gateway latency in ms.
- **`/roll [dice]`** — roll dice in `NdM` notation (default `1d6`, e.g. `2d20`);
  validates input, replies with the rolls and total.
- **`/choose <options>`** — pick one option from a comma-separated list.

Non-functional:

- **Config via environment only**: `DISCORD_TOKEN` (required), `GUILD_ID` (optional —
  instant command sync to one guild during development). `.env` supported locally,
  never committed.
- **Logging** to stdout (the platform's monitor will consume it in phase 1).
- **Errors**: user-facing failures reply with a short message; command logic is pure
  functions, unit-tested without network.
- **CI**: commitlint + pytest on every PR (`tests` job).

## Maintenance log

Chance/game commands added after the v1 baseline, each following the pure-logic +
thin-wrapper + `register` pattern established above:

- **`/casino`** (#3) — spins a three-reel one-armed bandit. Draws three symbols
  independently from a fixed 5-symbol set (`🍒 🍋 🔔 ⭐ 💎`) and replies
  with the reels plus one outcome tier: **jackpot** (three of a kind), **small win**
  (exactly two of a kind), or **bust** (all different). No bets, currency, balances,
  persistence, or per-user state — payouts remain a possible follow-up, out of scope.
- **Weighted reels** (#5) — reel draws are no longer uniform. Each reel draws
  independently from a fixed weight table (`CASINO_WEIGHTS`, strictly descending from
  `🍒` to `💎`), so common symbols dominate and a `💎` jackpot is a genuinely rare
  event. Outcome tiers and reply format are unchanged from #3.
